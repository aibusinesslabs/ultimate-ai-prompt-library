### P-01143 - Writing Clear and Concise Technical Specifications

**Category:** Writing & Content Creation  
**Subcategory:** Technical Writing  
**Difficulty:** Beginner  
**Tags:** `technical writing` `specifications` `clarity` `documentation`

#### Prompt:

**Task:**  
Create a clear, concise technical specification document for a software feature based on provided details.

**Context:**  
You are a technical writer tasked with drafting specifications that developers and stakeholders can easily understand and implement. The document should translate technical requirements into straightforward, actionable language.

**Input:**  
- **Feature Details:** {feature_name}, {feature_description} (name and detailed description of the software feature)  
- **Functional Requirements:** {requirements_list} (list of key functional requirements and behaviors)  
- **Constraints and Assumptions:** {constraints_assumptions} (any limitations, dependencies, or assumptions influencing the feature)  
- **Target Audience:** {audience_type} (description of the intended readers, e.g., developers, product managers)

**Instructions:**  
1. Begin with a brief introduction summarizing the feature purpose.  
2. Clearly outline each functional requirement in a bullet-point format.  
3. Specify any constraints or assumptions relevant to the implementation.  
4. Use simple, direct language suitable for the target audienceâ€™s technical background.  
5. Review for clarity, completeness, and logical flow before finalizing.

**Constraints:**  
- Length: 150-300 words  
- Format: Use bullet points for requirements and constraints sections  
- Style: Professional, clear, and jargon-free where possible  
- Structure: Introduction, Functional Requirements, Constraints and Assumptions, Summary

**Output Format:**  
A well-structured technical specification document in plain text following the outlined sections.

#### Use Case:  
This prompt is ideal for beginner technical writers or product managers needing to produce clear specifications for new software features. It ensures documentation that facilitates smooth communication between stakeholders and development teams.
